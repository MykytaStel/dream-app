# DESIGN.md

## Design Goal

The design of **Kaleidoscope of Dreams** should feel:
- atmospheric
- elegant
- intimate
- calm
- modern
- premium without being flashy

The interface should support moments when users are sleepy, emotionally open, or trying to remember fragile details.

---

## Core Design Principles

### 1. Calm Over Noise
Use restraint. Avoid visually loud UI.

### 2. Fast Capture First
The primary flow must minimize friction.

### 3. Emotional Warmth
The product should feel soft and human, not sterile.

### 4. High Readability
Users may use the app early in the morning or at night.

### 5. Depth Without Clutter
Advanced tools should not overwhelm the main flow.

---

## Visual Direction

### Tone
- moody but clear
- dreamlike but not cliché
- premium but not luxurious to the point of distance
- reflective, not corporate

### Avoid
- overly mystical stereotypes
- cheap “galaxy” aesthetics
- neon overload
- childish fantasy visuals
- too many colors competing at once

---

## Color Strategy

Recommended direction:
- dark-friendly palette
- muted accent colors
- soft gradients used sparingly
- strong contrast for legibility
- one or two signature accent tones

Possible palette mood:
- deep midnight tones
- soft plum / indigo / dusty violet accents
- muted warm neutrals for cards or highlights
- subtle moonlight-style contrast for key actions

---

## Typography

Typography should carry much of the premium feeling.

Guidelines:
- strong hierarchy
- clear distinction between title, body, metadata, and labels
- comfortable paragraph spacing
- avoid decorative fonts in dense reading areas
- expressive display font can be used sparingly for hero moments

Recommended structure:
- one primary readable sans-serif
- one optional accent display font for brand moments only

---

## Spacing and Layout

The product should breathe.

Guidelines:
- generous padding on key screens
- avoid cramped input forms
- use spacing to create emotional calm
- group related content clearly
- avoid overusing dividers if spacing can solve the problem

---

## Component Direction

### Core Components
- primary button
- secondary button
- input
- text area
- tag chip
- mood selector
- symbol chip
- journal card
- insight card
- bottom sheet
- modal
- empty state
- section header

### Component Rules
- rounded but not cartoonish
- tactile and touch-friendly
- consistent shadows and elevation
- consistent pressed states
- subtle animation feedback

---

## Motion Principles

Motion should feel intentional and soft.

Use motion for:
- screen transitions
- card reveal
- save confirmation
- filter interactions
- reordering or appearance of dream metadata

Avoid:
- heavy decorative animation
- long delays
- motion that blocks fast writing
- too many simultaneous moving elements

---

## Core Screen Intent

### Home
Should feel welcoming, gentle, and immediately useful.
Include:
- quick add action
- recent dreams preview
- calm insight snippet or mood snapshot
- lightweight reminder of value

### Add Dream
This is the most important screen.
It should be:
- distraction-free
- quick to scan
- easy to save
- forgiving if the user only remembers fragments

### Journal / Timeline
Should feel like a personal archive.
It should encourage revisiting without feeling like a spreadsheet.

### Insights
Should feel rewarding and understandable.
Not overly analytical.

### Settings
Should be simple and private.
Avoid making settings feel technical unless needed.

---

## Empty State Guidance

Empty states should be emotionally supportive, not generic.

Good empty state tone:
- quiet
- encouraging
- minimal
- slightly poetic if it matches the brand

Avoid:
- startup clichés
- cheesy motivation
- too much text

---

## Onboarding Guidance

Onboarding should be light and elegant.

Goals:
- explain core value quickly
- reduce hesitation
- show that the app is private and easy to use
- get users to first entry fast

Do not overload onboarding with too many features.

---

## Accessibility Considerations

The product should still be accessible despite its atmospheric style.

Ensure:
- readable contrast
- tappable touch targets
- clear text sizing
- support for dynamic type where feasible
- reduced motion support if animations become richer later

---

## Design System Recommendation

Create a lightweight design system with:
- color tokens
- spacing tokens
- radius tokens
- typography scale
- shadow/elevation tokens
- animation timings
- reusable surface patterns

Keep it flexible, not overengineered.

---

## Design Success Test

The design is working if users feel:
- safe writing personal dreams
- invited to return
- impressed by the polish
- calm while using the app
- that this app is more special than a generic journal
