# MVP-ROADMAP.md

## Roadmap Goal

This roadmap is designed to avoid building a shallow app.
It focuses on creating a product that can actually become distinctive, retain users, and later expand into a serious premium product.

---

## Guiding Strategy

Build in this order:
1. core capture experience
2. revisit value
3. retention loop
4. trust and privacy
5. sync and scale
6. insights and premium depth
7. AI and advanced differentiation

If the app is not great at the first two layers, advanced features will not save it.

---

## Phase 0 — Foundation

### Goals
- define product direction
- lock design principles
- set architecture direction
- create repo structure
- choose persistence approach

### Deliverables
- project docs
- navigation skeleton
- theme tokens
- base component library
- local storage foundation
- analytics event planning
- crash/error tooling decision

---

## Phase 1 — Core MVP

### Goal
Build the fastest, most satisfying dream capture experience possible.

### Features
- onboarding
- create dream
- edit dream
- delete dream
- title + body
- mood selection
- tags
- dream type
- favorite entry
- timeline / journal list
- basic search
- basic filters
- reminder notifications
- local persistence

### Success Criteria
- dream saved in under 30 seconds
- zero confusion in add flow
- journal feels usable from week one
- app works reliably offline

---

## Phase 2 — Stronger Retention MVP

### Goal
Give users reasons to revisit, not just record.

### Features
- calendar view
- recurring symbols
- mood overview
- insight preview cards
- better empty states
- more polished home screen
- quick-entry templates
- voice-to-text capture
- biometric lock

### Why This Phase Matters
Without revisit value, the app stays a one-time novelty.

---

## Phase 3 — Differentiation Layer

### Goal
Make the app feel clearly better than simple journal competitors.

### Features
- pattern highlights
- recurring symbol tracking
- emotional trends
- dream streaks without aggressive gamification
- personalized reminders
- visual dream cards
- widgets
- premium theme options

### Product Impact
This is where the product begins to feel richer, more alive, and more ownable.

---

## Phase 4 — Cloud and Account Layer

### Goal
Add safety, portability, and premium readiness.

### Features
- optional account creation
- cloud backup
- sync across devices
- export options
- account and device management
- remote config / feature flags

### Important Rule
Cloud should enhance the product, not become a hard dependency for core journaling.

---

## Phase 5 — Premium and Monetization Layer

### Goal
Introduce paid value that feels meaningful and non-exploitative.

### Features
- advanced insights
- premium themes
- advanced exports
- private lock enhancements
- widgets and visual packs
- extended history filters
- premium dream boards

### Monetization Rule
Never lock basic trust, journaling, or dignity behind a paywall.

---

## Phase 6 — AI Layer

### Goal
Use AI only where it creates clear value.

### Features
- AI summaries
- symbol extraction
- pattern detection
- dream recap generation
- interpretation support with careful tone

### Caution
AI should support the user, not pretend to be mystical truth.

---

## Recommended Build Order

### Sprint Sequence
1. design tokens and app shell
2. add dream screen
3. local persistence
4. journal list and dream detail
5. edit/delete/favorite
6. onboarding
7. search and filters
8. reminders
9. home screen polish
10. calendar and revisit features
11. biometrics and privacy polish
12. optional cloud and sync
13. insight cards
14. premium and AI experiments

---

## Anti-Roadmap Warning

Do not do these too early:
- complex backend
- social features
- heavy AI features
- overdesigned onboarding
- giant analytics systems
- excessive theming before core UX is strong

---

## Product Checkpoints

### Checkpoint A
Do users enjoy writing dreams here more than in Notes?

### Checkpoint B
Do users come back to browse old dreams?

### Checkpoint C
Do insight and personalization features make the app feel unique?

### Checkpoint D
Is there enough trust and delight to justify premium?

---

## Roadmap Summary

The winning path is:
- make recording delightful
- make revisiting rewarding
- make patterns visible
- make privacy obvious
- then scale into premium and AI
